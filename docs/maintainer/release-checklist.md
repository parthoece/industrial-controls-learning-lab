# Release Checklist

- [ ] CI passes on supported Python versions
- [ ] Internal links and repository checks pass
- [ ] Changelog is updated
- [ ] Version is updated in `pyproject.toml`, package metadata, and `CITATION.cff`
- [ ] New labs include normal and failure tests
- [ ] Safety language is reviewed
- [ ] No generated databases, credentials, or large binaries are committed
- [ ] Release notes distinguish simulation from hardware-tested work
- [ ] Tag and GitHub release use the same version
