# Role Map: Fundamentals Before Specialization

The target jobs share a common base. Build that base first; specialize through the evidence you emphasize.

## Automatic control engineer

### Fundamentals to demonstrate before applying

- signals, units, sampling, and simple dynamic models
- open- versus closed-loop behavior
- stability intuition and time-response metrics
- discrete P, PI, and PID control
- saturation, anti-windup, noise, delay, and actuator limitations
- reproducible simulation and tuning experiments
- honest assumptions and model limitations

### Usually deepened on the job or in graduate study

- state-space control and observers
- system identification and frequency-response methods
- Kalman filtering
- robust, nonlinear, adaptive, and model-predictive control
- high-performance servo tuning, feedforward, and vibration suppression
- formal stability analysis

## PC-based motion-controller engineer

### Fundamentals to demonstrate before applying

- cyclic tasks, deadlines, latency, jitter, and deterministic state updates
- hardware-independent axis interfaces
- motion profiles, homing, limits, and command lifecycle
- interlocks, state machines, diagnostics, and recovery
- EtherCAT concepts and systematic network diagnosis
- CiA 402 state and fault concepts
- concurrency boundaries, bounded queues, logging, and automated testing

### Usually deepened on the job

- real EtherCAT master integration and conformance
- RTOS/kernel configuration, CPU isolation, and real-time networking
- distributed-clock implementation details
- electronic gearing, camming, interpolation, gantry, robotics, or CNC
- vendor-specific motion libraries and drive commissioning
- safety certification and hardware validation

## Smart manufacturing software engineer

### Fundamentals to demonstrate before applying

- factory information layers and OT/IT boundaries
- OPC UA, MQTT, and REST use cases
- event, alarm, recipe, asset, cycle, and part-traceability models
- relational and time-series storage fundamentals
- OEE assumptions and calculations
- API contracts, observability, configuration, testing, and deployment basics
- least privilege, credential hygiene, segmentation, audit trails, and backups

### Usually deepened on the job

- enterprise MES/ERP integration
- highly available streaming and data platforms
- site-specific ISA-95 or regulated-industry models
- fleet and edge orchestration
- advanced analytics and predictive maintenance
- plant-wide cybersecurity programs

## Portfolio positioning

Use the same learning evidence differently:

- **Control roles:** lead with models, response metrics, tuning choices, disturbances, and limitations.
- **Motion roles:** lead with cyclic architecture, homing, drive states, command lifecycle, interlocks, and troubleshooting.
- **Manufacturing roles:** lead with data models, traceability, SQL, OEE, integration boundaries, observability, and deployment.
