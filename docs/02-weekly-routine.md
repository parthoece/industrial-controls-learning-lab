# Weekly Learning and Evidence Routine

## Monday — concept

Write a short explanation in your own words. Include units and a diagram when they reveal assumptions.

## Tuesday — references

Prefer official vendor, standards-body, or primary technical sources. Record:

- exact page and access date
- key idea
- what is general versus vendor-specific
- what remains uncertain

## Wednesday — implementation

Create the smallest program, state machine, schema, or diagram that exposes the concept. Keep it small enough to explain line by line.

## Thursday — test and diagnose

Run:

1. a normal case
2. a boundary case
3. a failure case

Record measurements and reason codes. “It looks correct” is not sufficient evidence.

## Friday — consolidate

Finish:

- one weekly note
- one code, diagram, test, measurement, or troubleshooting artifact
- one concise progress update
- the progress-tracker update

## Monthly review

Every four weeks, remove weak or duplicated notes, fix broken links, add missing tests, and update the root README with the best evidence.
