# Module 0 review — CS to controls

Answer without opening the weekly lessons.

1. Trace one production instruction to physical motion and back to a stored event.

<details><summary>Answer</summary>

Include operations intent, controller validation, network/device command, mechanics, feedback, and event record.

</details>

2. Give three reasons a command can be accepted but the physical result can fail.

<details><summary>Answer</summary>

Examples: drive fault, mechanical jam, stale or failed feedback.

</details>

3. What metadata makes a numeric value usable?

<details><summary>Answer</summary>

Unit, coordinate/frame, valid range, timestamp/freshness, quality, and source.

</details>

4. Why is ordinary interlock logic not functional safety?

<details><summary>Answer</summary>

It is not necessarily independently designed, safety-rated, validated, or certified for risk reduction.

</details>
